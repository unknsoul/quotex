2. ROOT CAUSE — WHY IT GOES OPPOSITE TO TRENDS

2.1  The Triple Barrier Labeling Problem

The system uses Triple Barrier labeling in triple_barrier_labeler.py with these settings: TP = 1.5 × ATR, SL = 1.0 × ATR, max_bars = 4 (20 minutes). This is the single largest cause of the opposite-direction problem.

2.2  The three_candle_momentum Feature Amplifies This

In feature_engineering.py at line 309, the feature three_candle_momentum is encoded as: d2 * 4 + d1 * 2 + d0. When 3 consecutive candles are green, this equals 7 (binary 111) — the maximum value. The model has learned from Triple Barrier labels that value=7 (three greens) precedes label=0 (SL hit). So the model correctly learned a wrong pattern — it is internally consistent but externally wrong.

2.3  Direction Flip Filter Worsens It (Strategy 5)

In telegram_bot.py (around line 706), Strategy 5 blocks any signal where the direction changes from the previous bar:


This filter was designed to avoid whipsaw entries. However, it also blocks the model from correctly switching direction at the start of a new trend. When the model finally detects an uptrend and signals UP after a series of DOWN signals, Strategy 5 blocks it as a "direction flip". The system is structurally incapable of catching trend reversals.
