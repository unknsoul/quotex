1.  SYSTEM OVERVIEW

1.1  What This System Is

QUOTEX LORD is a regime-aware, multi-model ML trading signal engine. It predicts the direction of the next 5-minute forex candle using a 5-seed XGBoost ensemble, a LightGBM meta-model, and a logistic weight learner — all trained with zero look-ahead leakage via purged walk-forward cross-validation. Predictions are delivered to Telegram subscribers with Kelly-sized position guidance, real outcome tracking, and automated daily/weekly reports.

1.3  Real Limitations (Acknowledged)

Data ceiling — OHLC candles are lossy. Institutional systems use tick data, order book imbalance, and liquidity depth. Without these, 90%+ accuracy is unrealistic.

Execution gap — The system models spread cost (3.5 pip) and 1-tick slippage but not fill latency or liquidity gaps. This matters for real-capital trades.

Feature ceiling — All 66 features derive from price. Professional systems also use news sentiment, macro calendars, and cross-asset flows. These are future upgrades.

OHLC data limitation is the hard floor. Everything else in this plan maximises performance within that constraint.

2.3  Confidence Assembly Formula

# Step 1 — Ensemble probability + uncertainty

all_probs  = [model.predict_proba(X)[0][1] for model in ensemble]  # 5 values

green_p    = mean(all_probs)

variance   = var(all_probs)

norm_var   = min(variance / 0.25, 1.0)   # normalise to [0,1]



# Step 2 — Meta reliability (+ online learner blend after 30 updates)

meta_rel   = meta_model.predict_proba(meta_row)[0][1]

if online_learner.n_updates >= 30:

    blend    = min(0.20, n_updates / 500)  # ramps from 0 to 20%

    meta_rel = (1 - blend) * meta_rel + blend * online_learner.predict(meta_row)



# Step 3 — Weight learner blend

primary_strength = abs(green_p - 0.5) * 2

weighted_score   = weight_model.predict_proba([[primary_strength, meta_rel,

                       regime_strength, 1.0 - norm_var]])[0][1]



# Step 4 — Final confidence with all adjustments

confidence = weighted_score * (1.0 - norm_var)   # penalise uncertainty

           * session_mult                          # 0.80 – 1.00 by session

           * (0.5 + 0.5 * regime_stability)       # penalise transitions

GAP 4 — Drift & Covariate Shift Not Running in Live Loop

🟠 HIGH — P1



⚠  drift_detector.py and covariate_shift_monitor.py are both complete but never called. The system has no awareness of feature distribution shifts that degrade model quality before accuracy visibly drops.



Files to change: auto_learner.py + telegram_bot.py

Step 1 — Add check_drift_triggers() function to auto_learner.py:



# ─── ADD: imports at top of auto_learner.py ─────────────────

from drift_detector import check_drift_from_files

from covariate_shift_monitor import check_covariate_shift



# ─── ADD: new function in auto_learner.py ────────────────────

def check_drift_triggers(symbol, recent_df=None, feature_names=None):

    '''Run PSI + KS-test drift checks. Returns list of trigger strings.'''

    triggers = []



    # Check 1: Feature distribution drift vs OOF baseline

    try:

        result = check_drift_from_files(

            train_path=OOF_PREDICTIONS_PATH,

            recent_df=recent_df, feature_names=feature_names)

        if result.get('overall_drift'):

            triggers.append(f'feature_drift: {result["n_drifted"]} features')

    except Exception as e:

        log.warning('Drift check failed: %s', e)



    # Check 2: Covariate shift (PSI per feature vs training baseline)

    if recent_df is not None and feature_names is not None:

        try:

            from data_collector import load_csv

            baseline = load_csv(symbol, 'M5')

            if baseline is not None and len(baseline) > 1000:

                shift = check_covariate_shift(

                    baseline_df=baseline.tail(2000),

                    current_df=recent_df.tail(500),

                    feature_cols=[f for f in feature_names if f in baseline.columns])

                if shift.get('retrain_needed'):

                    triggers.append(f'covariate_shift: {shift["n_warnings"]} features PSI>0.20')

        except Exception as e:

            log.warning('Covariate shift check failed: %s', e)



    return triggers

    while True:

        try:

            now = datetime.now(timezone.utc)

            days_left = (6 - now.weekday()) % 7

            if days_left == 0 and (now.hour > 21 or

                    (now.hour == 21 and now.minute >= 15)):

                days_left = 7

            target = (now + timedelta(days=days_left)).replace(

                hour=21, minute=15, second=0, microsecond=0)

            await asyncio.sleep((target - now).total_seconds())



            await _send_admin_alert(bot,

                f'🔄 Weekly lite retrain starting for {DEFAULT_SYMBOL}...')



            result = {}

            def _run():

                from auto_learner import retrain_lite

                result['ok'] = retrain_lite(DEFAULT_SYMBOL, validate=True)

            t = threading.Thread(target=_run, daemon=True)

            t.start(); t.join(timeout=600)



            ok = result.get('ok', False)

            await _send_admin_alert(bot,

                f'{'✅' if ok else '❌'} Weekly retrain {'complete' if ok else 'FAILED'}.')

            await asyncio.sleep(60)

        except Exception as e:

            log.exception('Weekly retrain job error: %s', e)

            await asyncio.sleep(3600)
